### Properties
<ResponseField name="identifier" type="string" required>
A unique identifier for the button. This is used to distinguish the button from others.
</ResponseField>
<ResponseField name="labels" type="{ [key in Locale]?: string }" required>
A dictionary of labels for the button, keyed by locale. This allows the button to display different text based on the user's locale.
</ResponseField>
<ResponseField name="fields" type="ButtonField[]" optional>
An optional array of fields that the button requires. Each field is defined by the ButtonField interface.
</ResponseField>
<ResponseField name="repositories" type="EntityClassOrSchema[]" optional>
An optional array of repositories that the button interacts with. Each repository is defined by the EntityClassOrSchema type.
</ResponseField>
<ResponseField name="execute" type="ButtonExecutor<InferOptions<T>, Repository<EntityInstanceType<R[number]>>[]>" required>
A function that is executed when the button is interacted with. This function receives the interaction, options, client, and repositories as parameters.
</ResponseField>
<ResponseField name="style" type="ButtonStyle" required>
The style of the button. This is defined by the ButtonStyle enum.
</ResponseField>
<ResponseField name="emoji" type="string" optional>
An optional emoji that is displayed on the button.
</ResponseField>

### Methods
<ResponseField name="constructor" type="Button<T, R>">
Initializes a new instance of the Button class with the provided options.
</ResponseField>

### Lifecycle
1. The `Button` class is instantiated with the required options.
2. The `ButtonManager` adds the button to manage its interactions.
3. The `execute` method is called when the button is interacted with, performing the associated action.

### Notes
<Note>
The Button component allows for the creation of interactive buttons in Discord messages, enhancing user engagement and interaction.
</Note>
<Note>
The Button component can be customized with different styles, labels, and actions, making it versatile for various use cases.
</Note>
<Warning>
Ensure that each button has a unique identifier to avoid conflicts and unintended behavior.
</Warning>
